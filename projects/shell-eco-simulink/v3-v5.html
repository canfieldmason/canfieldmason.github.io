Shell Eco-Marathon Simulation — Versions 3–5

Development of closed-loop driver control and realistic engine modeling.

Overview

Versions 3 through 5 introduced automated driver control and a realistic powertrain model for the Shell Eco-Marathon prototype vehicle.

These versions established the core simulation architecture used in later versions of the model.

Major Additions
  • Closed-loop driver control
  • Pulse-and-coast control concepts
  • Real dyno-based engine model
  • Torque vs RPM lookup tables
  • Vehicle resistive force modeling
  • Energy consumption modeling

Version 3 — PID Driver Control

Version 3 introduced a closed-loop driver model using a PID controller to maintain a target vehicle speed.

The controller adjusted throttle based on the difference between target speed and vehicle speed.

This version demonstrated stable vehicle behavior and provided a baseline for later control strategies.

Version 4 — Real Engine Model

Version 4 replaced the constant-power engine approximation with a realistic torque model derived from dynamometer testing data.

Engine torque was calculated from measured horsepower and engine speed and stored in a lookup table for use during simulation.

Figure V4-1 — Engine Torque Map

This version introduced physically realistic acceleration behavior and accurate engine operating regions.

Version 5 — Improved Driver Strategy

Version 5 refined the driver strategy by introducing speed bands and hysteresis to reduce throttle oscillations.

The controller attempted to maintain vehicle speed within a defined range around the target speed.

Figure V5-1 — Speed Control with Hysteresis

Limitations of Versions 3–5

  • Vehicle speed remained nearly constant
  • Throttle rarely reached zero
  • Engine operated continuously
  • Model behaved like cruise control
  • Did not match Shell Eco driving strategy

These limitations motivated the development of Version 6.
